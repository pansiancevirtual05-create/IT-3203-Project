HTML throughout the Years

The year of change for HTML came in 1995, as new updates emerged. This period saw the rise of the first major competition for the web, known as the Browser Wars. In the beginning, the popular browser that used HTML was Internet Explorer, but before Microsoft took over, Netscape Navigator was the popular way to get online, as it held 80% of the market share. They were the first to really push HTML to do cool things like display images and colors. However, Microsoft realized the web was the future and made Internet Explorer available for free with Windows 95 computers. This idea brought Netscape's market down. By 2003, Internet Explorer was on top of the market. The United States government became suspicious of Microsoft for the bundle tactic, arguing that because everyone used Windows and Internet Explorer, it would harm the market. This moment in history wasn't just about HTML; it was a massive legal battle and a large part of HTML history, because it showed that web browsers had become the most valuable real estate in the technology world. Even though Microsoft lost the lawsuit, it had won the war, and Internet Explorer remained on top for another decade. Although the competition was brutal, it was good for HTML because it forced the technology to get better and faster. As the web was being developed, web browsers looked up to the W3C [World Wide Web Consortium] to set the rules. The W3C was the main organization managing HTML. In the early 2000s, the W3C decided to stop working on HTML and focus on XHTML 2.0. They wanted the web to be strict and rigid like SGML, where a single error in the code would cause the website to crash and not run at all. This led to a rebellion. The browser makers, Apple, Mozilla, and Opera, were mad that the W3C moved very slowly and was heading in the wrong direction. They formed their own group called the WHATWG (Web Hypertext Application Technology Working Group) in 2004. The W3C wanted purity and order, while the WHATWG wanted a living language that could handle messy real-world apps, video, and audio.

One of WHATWG's biggest wins was the discontinuation of Adobe Flash Player. I remember for the longest time, while in Pre-K through 12th grade, if you wanted to watch a YouTube video or play online games in a browser, HTML couldn't do it; it’d say you needed to install Adobe Flash player. The WHATWG wanted HTML to be able to do these things without needing extra software. When HTML5 was released, it included tags like video and audio. This allowed developers to put media directly into the code. This proved that HTML was powerful enough to run full applications, not just static documents. The WHATWG began working on a specification called Web Applications 1.0, which eventually became HTML5. The W3C admitted defeat, abandoned XHTML, and adopted the WHATWG's HTML5 standard. It's why the web is so flexible today. Moreover, this shift to HTML5 wasn't just about discontinuing Adobe Flash; it was also about the web on mobile devices. In the late 2000s, the release of the iPhone changed everything. Old HTML websites looked bad on tiny screens because they were built for big desktop monitors. This was a problem in the market because would companies need to build a separate app for everything? HTML5 helped introduce the concept of Responsive Web Design. This allowed developers to write one single set of HTML code that could magically shrink or expand to fit a phone, a tablet, or a massive TV screen. This adaptability is the main reason why the web survived the App Store period. Instead of the web dying out in favor of native apps, HTML evolved to become flexible enough to live on any device. As technology continues to advance, HTML will continue to live and breathe; it’s going nowhere. We now have HTML 5.0, which will continue. There are some rumors about HTML 6.0, but the industry has moved to a Living Standard. Meaning, instead of waiting 5 years for a big update, the WHATWG updates HTML almost every day. It evolves constantly, just like an app on your phone that updates in the background without you noticing. This keeps the web modern and secure without breaking older websites. In the current web browser market, it seems Google Chrome has won. According to BrowserStack, Chrome holds roughly 65% to 70% of the global market share, while its closest competitor, Apple's Safari, sits around 15% to 20%. Because Chrome is so dominant, when Google adds new features to HTML, the rest of the web has to follow. A great example of these modern features is Hum to Search. Google Chrome and the Google App have added a feature where you can hum to find a certain song when you forget the words, but the rhythm is stuck in your head. This technology uses machine learning, where the browser listens to your voice through the microphone, permission granted via HTML API, and matches the melody to a database of millions of songs. This works very well, and I’m sure most people find it very useful, as I am one of them, and I like Google Chrome because of its features and how it’s built; it is very polished, easy to use, and has a great user interface.
